## Установка и развертывание

### Требования

- Веб-сервер (Apache, Nginx или IIS)
- Поддержка JavaScript (ES6 модули)
- HTTPS (рекомендуется)

### Apache

Проект использует `.htaccess` для маршрутизации. Убедитесь, что модуль `mod_rewrite` включен:

```apache
LoadModule rewrite_module modules/mod_rewrite.so
```

### Nginx

Используйте файл `nginx_rmwebsite.conf` как пример конфигурации. Скопируйте правила rewrite в ваш конфигурационный файл.

**Примечание**: Файлы `nginx_rmwebsite.conf` и `web.config` являются альтернативными конфигурациями и не используются одновременно с `.htaccess`. Используйте только конфигурацию для вашего сервера. Подробнее см. `CONFIG_FILES_README.md`.

### IIS

Используйте файл `web.config` для настройки URL Rewrite модуля.

**Примечание**: См. `CONFIG_FILES_README.md` для информации о том, какой конфигурационный файл использовать.

## Особенности

- **Service Worker**: Включен для кэширования ресурсов
- **Кэширование**: Настроено кэширование статических ресурсов (1 год для изображений, 1 месяц для CSS/JS)
- **Сжатие**: Включено gzip сжатие для текстовых ресурсов
- **Адаптивность**: Поддержка мобильных устройств и планшетов

## Известные проблемы и ограничения

### Критические проблемы, требующие решения:

1. **Внешние CDN зависимости**: Все JavaScript модули загружаются с внешнего CDN Readymag (`st-p.rmcdn1.net`). Для полной автономности необходимо:
   - Скачать все модули локально
   - Заменить импорты в файлах `dist/c/*.js` на локальные пути

2. **Большой размер HTML**: Файл `index.html` содержит встроенные данные ServerData (~200KB). Рекомендуется вынести в отдельный JSON файл.

3. **Base64 изображения**: HTML файлы в `snippets/` содержат base64-кодированные изображения. Рекомендуется вынести в отдельные файлы.
